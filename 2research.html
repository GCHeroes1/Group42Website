RCGP Toolkit Chatbot (COMP0073) - Julian-Ferdinand Vögele [1]

This project aims at building a chatbot to find relevant information, faster, and accurately. The RCGP has 27 toolkits, this project aims to build a chatbot that covers 2 of 27 toolkits.

RCGP toolkit chatbot is classified as an information chatbot, but no chatbot exists that fulfills what the RCGP is looking for, because the information basis of chatbot consists of the specific toolkits which are unique to RCGP.

Microsoft bot framework - comprehensive and easy-to-use SDK made of the framework itself and the bot connector SDK. allows for high degree of customisation, many integration possibilities, simple testing in the emulator. Range of predefined, customisable classes and functions. Even for the simplest logic a lot of over-complex code is needed. This difference is substantial given the vast amount of information, and the complex conversation flow of the RCGP toolkit chatbot.

Microsoft Health Bot Service - easy-to-use visual coding editor allowing developers to manage conversation flows and scenario interactions. Custom code can be added through action fragments. Highly maintainable and easily extensible. Solution aligns with the highest compliance and security standards and has a sound track record in health.

MHBS was chosen based on thorough comparison fpr the RCGP Toolkit Chatbot.

Cognitive services need to be selected to process natural language queries via API. enable the integration of state of the art AI based features. MHBS is the chatbot framework, Microsofts Language Understanding Intelligent Services (LUIS) and QnA maker as two cognitive services had a strong natural advantage over other solutions due to their smooth integrability with the framework. 2 areas for cognitive services were required, disambiguation and FAQ handling

LUIS was considered for disambiguation, as a machine learning based service, it integrates natural language understanding into chatbots matching user utterances with intents and entities. LUIS models are trained through a user interface and user query logs are used for active learning. Allows correction of model’s language interpretation and constantly improve the models. LUIS has a high performance, intuitive use, integrability advantage.

QnA maker was considered for FAQ handling. Cloud based API service integrates knowledge bases into chatbots. Knowledge bases are composed of question and answer pairs, can be built using semi-structures such as FAQs. given RCGP’s toolkits as the data input, the requirement for extensibility, the lack of competitors, and its smooth integration with the framework. QnA maker was chosen.

UCL’s agreement with microsoft to provide paid subscriptions and the fact that the cognitive services and the MHBS are azure services were strong reasons to use Azure for web hosting. RCGP IT team had some experience with azure. The web app for the chatbot was deployed using azure.
