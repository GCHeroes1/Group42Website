Final prototype

As a chatbot, the fundamental idea behind the chatbot remained the same- an IM interface that a GP can message into to ask questions. The difference in our prototypes is how they are accessed shown below; our first prototype stores the chatbot in a standalone application that must be downloaded. The second prototype is a web app which is accessed through a login screen.

Prototype 1 - Standalone application

Users are required to download and install the chatbot as an application on their device. They do this from the RCGP website.

Upon installation, they can open the application and login. If they don’t have a login, there is an option to register which opens the register account web page in a browser on the RCGP website.

Prototype 2 - Web application

New users are required to register first at the page on the left where they put in their NHS email and create a password for their account.

If they have an account already, they sign in using the screen on the right. This then takes you to the chatbot screen on the next slide.

Design principles

Design Principle Problem Solution Severity
Constraints Prototype 1- As the chatbot is a downloadable application, it can only be used on computers with it installed. Prototype 2 has less constraints as it is a web app and only requires an internet connection. A web app would be more suited and as a result we decided that this was the best way to move forward considering that mobility for GPs is important and they can’t always rely on software being installed. 2
Affordance To send a message, the user has to know to type and then click the enter button on their keyboard. But that might not be obvious and furthermore, they might be on a mobile device where it isn’t intuitive to do that. Put a send button next to the message box. 1
Visibility Because the documents open side by side to the IM screen, the messages become compressed and hard to read. It is hard to keep text visible without distorting the scale and overlapping over the speech bubbles as seen in our images. Make the chatbot the whole of the screen initially and then when the user clicks to open a document, split the screen to include the document and the messages side by side. Allow both sides to be resized accordingly and a close button for the document. 1
