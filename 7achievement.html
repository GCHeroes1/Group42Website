List of known Bugs

Incorrect Parser Matches

When the parser matches the row names to the field names of the database, they are given a score for how similar they are to each other. Currently, even though it finds the best match for each database field, it doesn't have a threshold score for how similar they should be. This means that it might match up a field and row name with the highest score, but it may also be a really poor score i.e. they aren't actually remotely close but are being paired together because there isn't any better match. As an example:

"Unity Assets for Project could be given a best match with Proposed Project Description, but it obviously shouldn't be. Although this is really rare, if a client for the IXN form was to rewrite the form themselves, they could cause this bug because there are no other names to match with.

This bug will need fixing through testing, to see at what score we should apply a cut off and block a match.

Table Duplicates

If we push a project that has been pushed before, the database doesn't detect this and creates a duplicate entry. We need to develop a way of detecting a duplicate value before a push and execute an update query on the original value instead.
