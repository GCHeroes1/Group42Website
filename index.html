A Chatbot for Healthcare Project Mapping in the UK

Project abstract

Currently, the problem for entrepreneurs, academics and research institutes is that it is difficult to research whether a particular project has been created/developed as there is no centralised means of querying for information.

Our solution was to create a chatbot which has the ability to query a database of projects with specific keywords. We researched the integration of the Microsoft Bot Framework with our SQL database being hosted within azure.

We have a Python script which can parse IXN forms and populate the SQL database, we went on to create an admin panel to allow for the utilisation of this script by admin staff who need to add data to the database. We also currently have a runnable chatbot framework which allows one way data retrieval from the database in a concise manner so that people can now research without the need to filter through clutter.

The final impact of our project is that we now have a database to serve as the national database for the collation of project data. Any authorised admin can contribute to this collection of data, they will have an easy to use administrator panel which will assist them in adding information to the database, as well as updating the existing information. Our target audience; entrepreneurs, academics and research institutes, will be able to use our publicly available chatbot service in order to do research on any number of projects which are stored in the database. This will be an ever growing set of information and the capability of our chatbot will only grow as we gain more training data for our integrated Language Understanding Intelligent Service (LUIS). We will be able to provide users with more and more accurate and helpful search results, to any number of users of any scope, regardless of which sector they work in, our chatbot can serve as valuable research to anyone.

Key features

Adaptive cards: We have utilised adaptivecards.io design unique card designs to display data in a user friendly and intuitive way. We have found, through user testing, that the current design choice of our adaptive cards are the easiest and quickest way to gain relevant information on any number of categories.

IXN miner: We have created a program which takes in a .docx file as input and extracts all of the essential information from it and uploads it to a database hosted on azure. The IXN miner employs algorithms which check for duplication of fields and also finds the rows in the column which most match the column headings in the database.

Administrator panel: We have firstly created an ixn miner which parsers .docx IXN filse and extracts relevant information and uploads it to the relative databases. In order to make this parser more easily usable by someone who will actually need to use it, we designed an administrator panel using tkinter, this panel allows for step by step guidance for adding new projects to the database and also updating existing information.

Luis machine learning: We have integrated luis with our project in order to parse the users messages and determine the intent of their message as well as the ability to identify keywords in their messages that would aid in construction of SQL queries so that the most relevant results can be displayed with absolute minimised clutter and ease of use.

Fuzzy string matching: We have utilised the "fuzzywuzzy" python library in order to aid in the parsing of the IXN forms incase fields were mispelt, as mistakes can happen with the best of us, but that doesnt mean that they should be made public on the database, so we do our best to essentially spell check our users.

Microsoft bot Framework: We chose to use the Microsoft Bot Framework to host our chatbot as it was a comprehensive and easy-to-use SDK (Software Development Kit) comprised of the framework itself and also the bot connector SDK. The Microsoft Bot Framework allows for a high degree of customisation, many integration possibilities, developer friendly testing with an emulator. There were also a range of predefined, customisable classes and functions. The downside of using this framework is that it could take a lot of over-complicated code for even the simplest of logic. Having worked with the Microsoft Bot Framework we do not regret our choice as it allowed for a very large amount of customisation. We especially liked the ability to integrate our own adaptive card templates.
